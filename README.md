# Async API Gateway with Adaptive Load Balancing

Асинхронный шлюз агрегации данных с динамической балансировкой нагрузки.

## Проект реализует:
- асинхронную агрегацию HTTP API,
- несколько стратегий параллельных запросов,
- адаптивную балансировку нагрузки,
- локальные API-эмуляторы,
- автоматическую систему тестирования,
- генерацию графиков и отчётов.

## Используемые технологии

- Python 3.10+
- FastAPI
- aiohttp
- asyncio
- pandas
- matplotlib
- uvicorn

## Запуск программы

Для корректной работы нужно запустить три отдельные терминала и в каждом выполнить запуск отдельной части программы:

### 1. Запуск API Gateway
```
uvicorn app.main:app --reload
```

Для проверки можно перейти в SwaggerUI по адресу <http://127.0.0.1:8000/docs>

### 2. Запуск эмуляторов
```
python emulators/run_all.py
```

Для проверки можно перейти по аддресам:
- <http://127.0.0.1:9001/data>
- <http://127.0.0.1:9002/data>
- <http://127.0.0.1:9003/data>

### 3. Запуск тестов
Запуск всех тестов 10 раз:
```
python main.py test --scenario all --repeats 10
```

Запуск отдельных тестов:
```
python main.py test --scenario <Название теста (A/B/C)> --repeats <Количество запусков>
```

После отработки тестов появятся файлы. В которых сохраняется информация о последнем запуске тестов.
```
reports/
│
├── report.json
└── report.csv
```

## Формирование графиков
Для формирования графиков необходимо запустить 
```
python visualization/generate_all.py
```

После генерации появятся графики
```
reports/graphs/
│
├── avg_time_comparison.png
├── adaptive_concurrency_C.png
└── success_vs_failed.png
```
### Описание графиков

```
avg_time_comparison.png
```
Линейный график:
- сценарии A/B/C,
- сравнение стратегий,
- сравнение среднего времени выполнения.

```
adaptive_concurrency_C.png
```
График изменения concurrency:
- показывает динамическую адаптацию нагрузки,
- только adaptive стратегия,
- только сценарий C.

```
success_vs_failed.png
```
Столбчатая диаграмма:
- сравнение стратегий,
- успешные запросы,
- неуспешные запросы.
